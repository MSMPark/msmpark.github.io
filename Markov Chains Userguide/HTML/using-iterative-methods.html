To understand this method, you need to understand a property of transition matrices. The following explanation covers this property.

Transition Matrix properties

You are given a certain number of states (Si) from which to choose in a transition matrix (A) with each entry being the conditional probability of transitioning from one state to another.

P(Si|Sj)=Aij.

To track multiple transitions, find the product of the two conditional probabilities. The following covers the scenarios in which the state starts in Si, and after two transitions ends in Si.

Si -> Sj -> Si:P(Sj|Si)*P(Si|Sj)=Aji*Aij.

Si -> Sk -> Si:P(Sk|Si)*P(Si|Sk)=Aki*Aik.

Si -> Si -> Si:P(Si|Si)*P(Si|Si)=Aii*Aii.

The probability of the event occurring is the sum of the three probabilities. Notice that this is identical to taking the ith row of the transition matrix and multiplying it by the ith column of the transition matrix-- the same operation that occurs with matrix multiplication. If you extrapolate this observation, you can track the probability of ending on a certain state after a certain number of transitions. To do this, raise the transition matrix to the power of the number of transitions.

Creating an iterative method

Now that you know the property of transition matrices, you can find how many steps it takes to reach the steady state and what the steady state is. The following method is simple, and follows two steps:

  1. Sets a margin of error.

  2. Checks to see if the difference between a transition matrix raised to a power and the same matrix raised to the power + 1 is less than the margin of error.

    1. If it is, then the programs stops iterating.

    2. If it isn't, the logic check is repeated with the next power.

The number of times this loop iterates is the number of transitions needed to reach the steady state. The steady state is the transition matrix raised to the number of transitions.
